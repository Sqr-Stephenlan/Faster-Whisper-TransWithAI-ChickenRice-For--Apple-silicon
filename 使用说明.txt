基本用法:

将需要转录/翻译的音频或视频文件（或包含它们的文件夹）拖放到相应的批处理文件上运行。

=== macOS Apple Silicon CPU ===

适用范围：Apple Silicon（arm64）Mac 源码运行。主模型使用 CTranslate2 CPU int8；不使用 Metal、MPS、CoreML 主模型加速或 Apple Neural Engine。VAD 仅使用 ONNX Runtime CPUExecutionProvider。

1. 环境安装与检查

在终端进入仓库目录后运行：

  ./dev.sh doctor

项目已有可用 .venv 时会直接使用，禁止删除或重建。若新机器没有 .venv，在确认要创建环境后运行：

  ./dev.sh bootstrap

运行依赖来自 requirements-macos.txt，并受 constraints-macos-arm64.txt 的已验证版本约束；测试和静态检查依赖在 requirements-dev.txt。Mac CPU 运行时不需要 Modal、PyInstaller、CUDA 或 ROCm。

2. 模型准备

下载前请检查可用磁盘空间。模型较大，不进入 Git。首次下载需要网络：

  ./dev.sh python download_models.py --profile translate --non-interactive
  ./dev.sh python download_models.py --profile transcribe --non-interactive

也可一次准备全部资产：

  ./dev.sh python download_models.py --profile all --non-interactive

固定目录：

  models/whisper_vad.onnx
  models/whisper_vad_metadata.json
  models/whisper-base/preprocessor_config.json
  models/whisper-base/config.json
  models/translate/       日译中模型
  models/transcribe/      日文转录模型

翻译和转录是不同的 CT2 模型。仅把 --task 从 translate 改成 transcribe 不会把翻译模型变成转录模型，必须同时选择对应模型目录。

严格检查（可双击“检查Mac环境.command”）：

  ./dev.sh python scripts/macos_doctor.py --mode all --json

缺失、空文件、非法 JSON、ONNX 无法加载或 provider 错误都会返回非零退出码。只检查、不联网：

  ./dev.sh python download_models.py --profile all --verify-only --non-interactive

3. Finder 双击与终端运行

双击“运行(翻译)(CPU).command”或“运行(转录)(CPU).command”，在原生选择器中多选文件。取消选择会正常退出，不启动模型。也可在终端传入单个文件、多个文件或文件夹；空格、中文和括号路径会保持为独立参数。

  './运行(翻译)(CPU).command' '/Users/me/日语 音频（测试）/第 1 集.m4a'
  './运行(转录)(CPU).command' '/Users/me/第 1 集.m4a' '/Users/me/第 2 集.m4a'

完全展开的翻译命令：

  ./dev.sh python infer.py --model_name_or_path "$PWD/models/translate" --task translate --device cpu --compute_type int8 --cpu_threads 12 --vad_threads 4 --audio_suffixes "mp3,wav,flac,m4a,aac,ogg,wma,mp4,mkv,avi,mov,webm,flv,wmv" --sub_formats "srt,vtt,lrc" "/Users/me/日语 音频（测试）/第 1 集.m4a"

默认生成 SRT、VTT、LRC。已存在的全部目标字幕且未指定 --overwrite 时会跳过并返回 0。可用 --overwrite 重做，用 --output_dir 指定含空格或中文的输出目录。

4. 离线与日志

首次安装和下载需要网络。doctor 通过后，推理设置 HF_HUB_OFFLINE 和 TRANSFORMERS_OFFLINE，并仅加载本地 whisper-base、VAD 和主模型，可断网运行。模型缺失或损坏不会静默联网，也不会写空字幕冒充成功。

详细日志位于仓库根目录 latest.log。报错时请收集：

  - latest.log
  - ./dev.sh python scripts/macos_doctor.py --mode all --json 的输出
  - 实际命令（隐去私人路径）
  - 退出码：运行时/模型错误为 1，无有效输入为 2，成功或全部跳过为 0

5. 本地测试媒体约定（不得提交 Git）

  testdata/local/japanese-short.*   10–30 秒功能样本
  testdata/local/japanese-5min.*    性能样本

自动化测试如需音频，应使用标准库 wave 在临时目录生成静音或正弦 WAV，不提交二进制 fixture。

6. 最终验收清单

  [ ] ./dev.sh doctor、pip check、pytest、Ruff、Mypy、compileall 全部通过
  [ ] doctor 的 VAD provider 只有 CPUExecutionProvider
  [ ] 翻译短样本生成非空中文 SRT/VTT/LRC
  [ ] 转录短样本生成非空日文 SRT/VTT/LRC
  [ ] 多文件、目录、中文/空格/括号路径和中文输出目录通过
  [ ] 已存在字幕跳过、--overwrite 重做、纯静音日志语义通过
  [ ] 完整模型在断网环境仍可推理
  [ ] Finder 三个 command 可打开，取消无 traceback
  [ ] 5 分钟样本完成 cpu_threads/vad_threads 基准并记录选定值

没有主模型或真实日语样本时，只能标记“等待端到端资产”，不得宣称完整完成。

macOS 常见错误：

  - doctor 报 models/translate 或 models/transcribe 缺失：下载对应 profile，不能只改 --task。
  - command 无法打开：在终端运行 chmod +x *.command 后重试，并查看 macOS 安全性提示。
  - 没有字幕：先看 latest.log；若确为纯静音，允许生成空字幕，但日志应明确未检测到语音。
  - 模型/运行时错误：doctor 必须先通过；不要手工放置残缺模型文件。

=== 选择运行模式 ===

翻译模式:
- 拖放到 "运行(翻译)(CPU).bat" - 使用CPU进行日文转中文翻译
- 拖放到 "运行(翻译)(GPU).bat" - 使用GPU进行日文转中文翻译（显存≥6GB时使用）
- 拖放到 "运行(翻译)(GPU,低显存模式).bat" - 低显存GPU翻译（int8_float16）
- 拖放到 "运行(翻译)(GPU,高显存加速模式).bat" - 批处理加速翻译（建议8GB+显存）

转录模式:
- 拖放到 "运行(转录)(CPU).bat" - 使用CPU进行日文原文转录
- 拖放到 "运行(转录)(GPU).bat" - 使用GPU进行日文原文转录（显存≥6GB时使用）
- 拖放到 "运行(转录)(GPU,低显存模式).bat" - 低显存GPU转录（int8_float16）
- 拖放到 "运行(转录)(GPU,高显存加速模式).bat" - 批处理加速转录（建议8GB+显存）

GPU模式支持NVIDIA CUDA版本和AMD ROCm/HIP版本。建议先更新显卡驱动到最新版本。

输出到指定文件夹:
- 拖放到 "运行(翻译)(GPU)(输出到当前文件夹).bat" - 翻译字幕输出到"输出"文件夹
- 拖放到 "运行(转录)(GPU)(输出到当前文件夹).bat" - 转录字幕输出到"输出"文件夹

=== 支持的格式 ===

音频格式: mp3, wav, flac, m4a, aac, ogg, wma
视频格式: mp4, mkv, avi, mov, webm, flv, wmv

输出格式:
- LRC (歌词格式，适合音乐播放器)
- SRT (常用字幕格式，适合视频播放器)
- VTT (WebVTT格式，适合网页视频)

所有批处理文件默认会生成这三种格式的字幕。如果字幕文件已存在，将自动跳过。

无主模型版注意：默认批处理文件会从 models 文件夹根目录加载主 Whisper 模型。
- 如果不想修改批处理文件，请把 config.json、model.bin/model.safetensors 等模型文件直接放到 models 文件夹下。
- 如果模型保存在 models\模型文件夹\ 下，请在对应批处理文件的 infer.exe 后添加 --model_name_or_path="models\模型文件夹"。

调整参数:

1. 基本参数调整：
编辑批处理文件，找到以 "%cpath%\infer.exe" 开头的行，在其后添加参数。

示例（添加覆盖模式）:
添加前: "%cpath%\infer.exe" --device="cuda" %*
添加后: "%cpath%\infer.exe" --overwrite --device="cuda" %*

常用参数:
--overwrite : 覆盖已存在的字幕文件
--output_dir="路径" : 指定输出文件夹（默认输出到源文件所在文件夹）
--audio_suffixes="mp3,wav" : 自定义处理的文件格式
--sub_formats="srt,vtt,lrc" : 自定义输出格式
--log_level="INFO" : 减少控制台输出（默认为DEBUG）
--task="transcribe" : 覆盖Whisper任务（translate=翻译，transcribe=转录）

2. 生成参数调整（高级）:
编辑 generation_config.json5 文件调整转录参数。
可在 generation_config.json5 中设置 "task": "translate" 或 "task": "transcribe"。
命令行 --task 参数优先级更高，会覆盖配置文件中的 task。
参数详情见下方相关项目链接。

补充：字幕合并/去重（segment_merge）
- 程序会尝试合并一些重复/重叠的识别片段，以减少“嗯嗯啊啊”等导致的重复字幕。
- 如果遇到“某一句字幕持续时间特别长（跨很久静音也不消失）”的情况，可在 generation_config.json5 里调整：
  - segment_merge.enabled: 关闭/开启合并
  - segment_merge.max_gap_ms: 只有当相邻两段间隔很短时才允许合并（建议 500~2000）
  - segment_merge.max_duration_ms: 单条字幕合并后的最大时长（建议 15000~30000）

注意：通常不需要调整生成参数。如遇到以下情况可尝试调整：
- 声音过小导致漏翻
- 时间轴对不上
- 出现幻听

---

故障排除:

1. GPU模式无法运行：
   - 确认是否为NVIDIA显卡
   - 更新显卡驱动到最新版本
   - 检查CUDA是否正确安装

2. 字幕未生成：
   - 检查文件格式是否支持
   - 查看控制台是否有错误信息
   - 尝试使用 --overwrite 参数重新生成

3. 内存不足：
   - 使用低显存模式
   - 尝试CPU模式
   - 处理较小的文件或分段处理

=== Modal 云端推理模式 ===

如果本地没有GPU或显存不足，可以使用Modal云端GPU进行推理。

1. 环境配置：

使用 Conda 创建轻量级环境（仅需 modal 和 questionary 库）：

```bash
conda env create -f environment-modal.yml
conda activate faster-whisper-modal
```

或手动安装：
```bash
pip install modal questionary
```

2. Modal 账号设置：

(1) 注册 Modal 账号：
   访问 https://modal.com/ 注册账号
   新用户每月有 $30 免费额度，足够处理大量音频

(2) 配置 Modal Token：
   在终端运行：
   ```bash
   modal token new
   ```
   按提示完成浏览器授权

(3) 配置 HuggingFace Token（可选但推荐）：

   为什么需要配置：
   - Modal 需要从 HuggingFace 下载模型文件
   - 某些模型需要登录才能下载
   - 配置后可避免下载失败或速度慢的问题

   如何配置：
   a. 获取 HF Token：
      访问 https://huggingface.co/settings/tokens
      创建一个 Read 权限的 token

   b. 在 Modal 中添加 Secret：
      访问 https://modal.com/secrets
      点击 "Create new secret"
      选择 "Hugging Face" 类型
      输入你的 HF Token
      保存为 "huggingface-secret"

3. 运行 Modal 推理：

```bash
python modal_infer.py
```

程序会交互式询问：
- GPU 类型：T4（推荐）、A10G、A100、H100 等
- 模型选择：基础版、海南鸡（日文转中文优化）、TransWithAI 日文转录、自定义模型
- 输入文件：本地音频文件或文件夹路径
- 批处理选项：是否启用批处理加速（需要更高显存）
- 超时时间：任务超时时间（分钟）

处理流程：
1. 上传本地音频文件到 Modal Volume
2. 在云端 GPU 上运行推理
3. 自动下载生成的字幕文件到本地
4. 输出结果保存在源文件同目录

注意事项：
- 首次运行需要构建 Docker 镜像，可能需要几分钟
- 模型首次下载会缓存在 Modal Volume，后续运行更快
- 推荐使用 T4 GPU，性价比最高
- 批处理模式可显著加速，但需要更多显存
