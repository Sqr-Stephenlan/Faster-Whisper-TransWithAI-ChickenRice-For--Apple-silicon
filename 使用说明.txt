Faster Whisper TransWithAI ChickenRice for macOS 使用说明
==========================================================

适用范围
--------

本项目面向 Apple Silicon（arm64）Mac，从源码运行。主模型使用 CTranslate2 CPU int8；VAD 使用 ONNX Runtime CPUExecutionProvider。本地流程不依赖独立显卡。

一、检查与安装环境
------------------

进入仓库目录后运行：

  ./dev.sh doctor

项目已有可用 .venv 时会直接使用，不要删除或重建。如果新机器没有 .venv，在确认需要创建环境后运行：

  ./dev.sh bootstrap

运行依赖来自 requirements-macos.txt，并受 constraints-macos-arm64.txt 的已验证版本约束；开发检查依赖位于 requirements-dev.txt。

二、准备模型
------------

首次下载需要网络和足够磁盘空间：

  ./dev.sh python download_models.py --profile translate --non-interactive
  ./dev.sh python download_models.py --profile transcribe --non-interactive

也可一次准备全部资产：

  ./dev.sh python download_models.py --profile all --non-interactive

模型固定目录：

  models/whisper_vad.onnx
  models/whisper_vad_metadata.json
  models/whisper-base/
  models/translate/       日译中模型
  models/transcribe/      日文原文转录模型

翻译和转录必须使用各自模型，不能只修改 --task 相互替代。

严格检查：

  ./dev.sh python scripts/macos_doctor.py --mode translate --json
  ./dev.sh python scripts/macos_doctor.py --mode transcribe --json

仅验证现有下载、不联网：

  ./dev.sh python download_models.py --profile all --verify-only --non-interactive

三、Finder 与终端运行
--------------------

Finder 双击方式：

  1. 双击“检查Mac环境.command”。
  2. 双击“运行(翻译)(CPU).command”或“运行(转录)(CPU).command”。
  3. 在原生选择器中选择一个或多个文件。

终端方式：

  './运行(翻译)(CPU).command' '/Users/me/日语 音频/第 1 集.m4a'
  './运行(转录)(CPU).command' '/Users/me/第 1 集.m4a' '/Users/me/第 2 集.m4a'

支持单文件、多文件和目录。空格、中文、括号路径会作为独立参数保留。默认输出 SRT、VTT、LRC；目标字幕均存在时会跳过。常用选项：

  --overwrite                 覆盖已有字幕
  --output-dir '/字幕目录'     指定输出目录

四、离线、日志与退出码
----------------------

模型完整后，Mac 启动器仅加载本地资产，可断网推理。模型缺失或损坏会在创建字幕前失败。

日志写入仓库根目录 latest.log：

  0  成功或全部跳过
  1  运行时或模型错误
  2  没有有效输入

纯静音允许生成空字幕，日志会明确显示未检测到语音。

五、Modal 云端推理（可选）
--------------------------

创建本地客户端环境：

  conda env create -f environment-modal.yml
  conda activate faster-whisper-modal

配置账号并启动：

  modal token new
  python modal_infer.py

远端容器使用 environment-modal-gpu.yml。运行会产生云端费用，请事先确认账号额度、GPU 类型和超时时间。

六、开发检查
------------

  ./dev.sh pip check
  ./dev.sh pytest
  ./dev.sh ruff check .
  ./dev.sh ruff format --check .
  ./dev.sh mypy --config-file pyproject.toml src infer.py download_models.py modal_infer.py scripts
  ./dev.sh python -m compileall -q infer.py download_models.py modal_infer.py scripts src tests

本地测试媒体、生成字幕、基准结果、latest.log、__pycache__ 和工具缓存均为临时数据，不得提交 Git。
